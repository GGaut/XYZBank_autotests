### Проект автоматизированного тестирования XYZBank

## Конвейер CI/CD

Проект для автоматизированного тестирования: используется GitLab CI/CD. Конвейер настроен для автоматического запуска тестов и создания отчетов о тестировании.

# Обзор конвейера

- **Среда**: Python 3.10
- **Менеджер пакетов**: UV
- **Браузер**: Google Chrome
- **Платформа тестирования**: pytest с параллельным выполнением

# Этапы конвейера

- Запускает все тесты параллельно
- Генерирует отчеты об испытаниях Allure
- Сохраняет результаты тестирования в виде артефактов

# Ключевые особенности

- Параллельное выполнение тестов для получения более быстрых результатов
- Автоматическая повторная попытка при сбое теста
- Кэширование зависимостей для ускорения сборки
- Генерация отчетов Allure
- Защита от 30-минутного тайм-аута

# Артефакты

- Отчеты о тестировании хранятся в разделе "allure-результаты"./`
- Отчеты доступны в течение 7 дней после запуска конвейера

## Тест-кейс 1: Создание клиента (Add Customer)

# Описание:

Проверка функциональности создания нового клиента с автоматически сгенерированными данными

# Предусловия:

Находиться на странице Manager
Раздел "Add Customer" доступен

# Шаги:

1. Перейти на вкладку "Add Customer"
2. Сгенерировать Post Code из 10 цифр
3. Преобразовать Post Code в First Name по алгоритму:

- Разбить Post Code на двузначные числа
- Каждое число преобразовать в букву английского алфавита (0=a, 1=b, ..., 25=z)
- Если число > 25, использовать остаток от деления на 26

4. Ввести сгенерированный Post Code в поле "Post Code"
5. Ввести преобразованное имя в поле "First Name"
6. Ввести сгенерированное значение в поле "Last Name"
7. Нажать кнопку "Add Customer"
8. Проверить появление сообщения об успешном создании клиента
9. Перейти на вкладку "Customers"
10. Проверить, что новый клиент отображается в таблице

# Ожидаемый результат:

- Клиент успешно создан
- Сообщение об успешном создании отображается
- Новый клиент присутствует в таблице Customers
- First Name соответствует алгоритму преобразования Post Code

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

## Тест-кейс 2: Сортировка клиентов по имени (First Name)

# Описание:

Проверка функциональности сортировки клиентов по имени в алфавитном порядке

# Предусловия:

Находиться на странице Manager
Вкладка "Customers" содержит как минимум 3 клиента с разными именами

# Шаги

1. Перейти на вкладку "Customers"
2. Проверить начальное состояние таблицы (не отсортирована)
3. Нажать на заголовок столбца "First Name" для сортировки
4. Проверить, что таблица отсортирована по возрастанию
5. Нажать на заголовок "First Name" еще раз
6. Проверить, что таблица отсортирована по убыванию
7. Проверить корректность сортировки всех записей

# Ожидаемый результат:

Таблица клиентов сортируется по имени при клике на заголовок столбца
Сортировка работает в обоих направлениях
Все записи корректно упорядочены

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

## Тест-кейс 3: Удаление клиента по условию

# Описание:

Проверка функциональности удаления клиента с именем, наиболее близким к средней длине имен

# Предусловия:

Находиться на странице Manager
Вкладка "Customers" содержит как минимум 3 клиента с разными именами

# Шаги

1. Перейти на вкладку "Customers"
2. Вычислить длины имен
3. Определить имя наиболее близкое к среднему значению
4. Удалить клиента с именем, наиболее близким к среднему значению
5. Проверить удаление и соответсвие условию

# Ожидаемый результат:

Клиент удален.
Удаленный клиент соответсвует заданному условию.

### Ручной запуск тестов

# Запуск с Allure отчетами и параллельными тестами:

"pytest --alluredir=./allure-results -q -n auto"

# Генерация и открытие Allure отчета

"allure serve ./allure-results"
